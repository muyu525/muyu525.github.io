Linux Container

Mon 19 January 2015 | tags: lxc

Linux Container简写LXC,是一种内核虚拟化技术，可以提供轻量级的虚拟化，以便隔离进程和资源，而且不需要提供指令解释机制以及全虚拟化的其他复杂性。

容器有效地将由单个操作系统管理的资源划分到孤立的组中，以更好地在孤立的组之间平衡有冲突的资源使用需求。

使用LXC可以方便的搭建多个虚拟主机，我主要用于模拟小型的集群，实践测试各个分布式应用。

下面记录一些简单的使用方法，详细介绍请搜索相关内容。

创建节点

以ubuntu为例

# Create a "p1" container using the "ubuntu" template and the same version of Ubuntu # and architecture as the host. Pass "-- --help" to list all available options.

# -t: 模板，LXC默认支持ubuntu, archlinux等多个linux发行版

# -n: 节点名称

sudo lxc-create -t ubuntu -n p1

运行节点

# Start the container (in the background)

sudo lxc-start -n p1 -d

连接节点

# Enter the container in one of those ways## Attach to the container's console (ctrl-a + q to detach)

sudo lxc-console -n p1

# Spawn bash directly in the container (bypassing the console login), requires a >= 3.8 kernel

sudo lxc-attach -n p1

# SSH into it

sudo lxc-info -n p1

ssh ubuntu@<ip from lxc-info>

停止节点

# Stop the container in one of those ways # Stop it from within

sudo poweroff

# Stop it cleanly from the outside

sudo lxc-stop -n p1

# Kill it from the outside

sudo lxc-stop -n p1 -k

以上只是简单介绍了LXC的几个常用命令，用于模拟集群测试已经可以满足需求。 当然，LXC还有很多特性，想要了解的可以查看官网: http://www.linuxcontainers.org
